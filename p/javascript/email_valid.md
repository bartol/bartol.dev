I am sorry, but this is a [thing](http://www.ex-parrot.com/~pdw/Mail-RFC822-Address.html). Of course, I am kidding. You don't have to use this gigantic block of regex. Here is simple one that will do it for virtually every use case. It's not prone to bugs where `@.@.@.@` is validated like some other short ones.

<!-- prettier-ignore -->
```js
/^[^\s@]+@[^\s@]+\.[^\s@]+$/
```

While this will do for mere mortals, there is other option for cool JavaScript kids.

## Let's increase our bundle size!

Yes, you got that right! Other opinion is to install package from npm!

```js
npm install validator
```

Congrats, your bundle will love it!

Also, make sure to import it like this so your fancy tree-shaker doesn't need to shake too much.

```js
import isEmail from 'validator/lib/isEmail'
```

And that's it. Who would say that you can validate email in _this_ many ways!
